# JavaScript Actions — Steering Guide

Use this when creating or modifying JavaScript actions via the MCP server.

> **Important:** Load the `javascript-action` MCP skill for the full reference before working with JS actions.

## File Location

JavaScript actions live at:
```
/jsactions/<module_name_lowercase>/actions/<ActionName>.js
```

Example: `/jsactions/myfirstmodule/actions/MyAction.js`

## MCP Tools

```
glob(pattern="/jsactions/myfirstmodule/actions/*.js")             → list actions
read_file(path="/jsactions/myfirstmodule/actions/MyAction.js")    → read action
write_file(path="/jsactions/myfirstmodule/actions/MyAction.js",   → write action
  newContent="...")
write_file(..., span={"startLine": 10, "endLine": 20})            → update range
```

The `ped_create_document` tool is also needed to register the action in the project model — get schema for `JavaScriptActions$JavaScriptAction` first.

## JavaScript Action Template

```javascript
// This file was generated by Mendix Studio Pro.
//
// WARNING: Only the following code will be retained when actions are regenerated:
// - the code between BEGIN USER CODE and END USER CODE
// Other code you write will be lost the next time you deploy the project.

/**
 * @param {string} inputParam - Description of the parameter
 * @returns {Promise.<string>}
 */
export async function MyAction(inputParam) {
    // BEGIN USER CODE
    // Your implementation here
    return inputParam;
    // END USER CODE
}
```

## Key Rules

1. **Only modify code between `BEGIN USER CODE` and `END USER CODE`** — everything outside is regenerated by Studio Pro
2. **Always return a Promise** — use `async/await` or return a `Promise` explicitly
3. **Return types must match** the declared return type in the action definition
4. **Import dependencies carefully** — only use packages available in the Mendix client runtime

## Available Existing Actions (reference)

From `nanoflowcommons`:
- `Base64Encode` / `Base64Decode` — encode/decode base64 strings
- `GenerateUniqueID` — generate a UUID
- `GetGuid` — get the GUID of a Mendix object
- `ShowProgress` / `HideProgress` — show/hide loading indicator
- `OpenURL` — open a URL in the browser
- `NavigateTo` — navigate to a page
- `Wait` — pause execution for N milliseconds
- `GetPlatform` — detect web/native platform

From `webactions`:
- `ScrollTo` — scroll to a widget
- `SetFocus` — focus a widget
- `ReadCookie` / `SetCookie` — browser cookie management

From `datawidgets`:
- `Export_To_Excel` — export data grid to Excel
- `Reset_Filter` / `Reset_All_Filters` — reset data grid filters
- `Set_Filter` — programmatically set a filter
- `Clear_Selection` — clear data grid selection

## After Writing

Verify the file was written correctly:
```
read_file(path="/jsactions/myfirstmodule/actions/MyAction.js")
```
